Architecture Deep Dive

How COLT works

A pipeline that transforms natural language into verified browser execution.

Phase 1

Discovery —

Crawls your application via Playwright, discovers every interactable element, verifies visibility via getBoundingClientRect() + CSS checks, generates natural language descriptions.

  • Finds input, button, a[href], ARIA roles, [data-testid], summary
  • Runs 7 hidden UI strategies (timed, keyboard, context menus, etc.)
  • Selector cascade: #id → [data-testid] → [name] → [aria-label] → [title]

COLT App

Context-Aware Code Generation

Describe your agent in natural language. The COLT App resolves pages and elements from the semantic index, generates standalone Python with real selectors and fallback chains.

  • Every selector comes from the discovered index — not placeholders
  • Exported agents are standalone — no COLT runtime needed
  • Inherits multi-strategy fallback chains and error handling
  • Drop into CI/CD, cron jobs, or your own codebase
Phase 5

Success Verification

After execution, COLT verifies task completion via 13 CSS selectors targeting common success patterns, plus Tailwind toast class detection. Catches both explicit confirmations and implicit state changes.
