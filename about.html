About COLT

Closing the gap between intent and execution

Browser agents should understand applications the way humans do — semantically, adaptively, and autonomously.

The Problem

Every browser automation tool today requires you to write explicit selectors and script every interaction. When the UI changes, the scripts break. When a modal appears, the scripts fail. When content loads dynamically, the scripts time out.

The gap between what you want an agent to do and what you have to code for it to work is enormous.

COLT eliminates that gap. It gives your agents a semantic understanding of any web application — what pages exist, what each element does, how to accomplish tasks — and an autonomous execution engine that adapts to live browser state.

The Approach

Discovery first. COLT crawls your application, discovers every interactable element, and indexes it semantically. This gives agents a map of your UI before they ever try to interact with it.

The execution engine implements a ReAct loop that observes live browser state, reasons about what to do, executes one action, and validates the result. It handles modals, loading states, errors, and recovery without pre-scripting.

The COLT App extends this by generating production-ready agent code that is fully context-aware. Describe what your agent should do, and the app produces standalone Python scripts with real selectors and fallback chains from the discovered UI. No COLT runtime required.

Design Principles

Discovery over assumption

Never assume the UI structure. Always crawl, discover, index, then act on what actually exists.

Autonomy with guardrails

The ReAct loop adapts in real time, but hard limits (25 iterations, 30 LLM calls) prevent runaway execution.

Resilience by default

Every interaction has a fallback chain. Every selector uses CSS.escape(). Nothing is fragile by design.
